## Phase Objectives
Finalize the global platform state, refine the UX, and prepare for backend RAG integration.

## Working (Mechanism)
1. **Global State**: Ensures Streaks, Badges, and Notes persist across different educational levels.
2. **RAG Readiness**: Implements API hooks and placeholder logic for NCERT document retrieval.
3. **Refinement**: Polishing animations, fixing minor UI glitches, and optimizing load times.

## Need (Rationale)
A professional product requires seamless transitions between its parts. Ensuring that achievements (Badges) and habits (Streaks) are globally recognized makes the platform feel like a single ecosystem. RAG readiness ensures that once the backend is ready, the UI can plug in immediately.
